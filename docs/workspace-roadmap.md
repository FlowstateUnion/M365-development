# Workspace Roadmap

Planned structure for M365 work under this repo:

## `projects/`

Use one folder per initiative, for example:

- `copilot-studio-sharepoint-logger`
- `teams-automation`
- `sharepoint-list-ops`
- `graph-experiments`
- `azure-runbooks`

## `runbooks/`

Use for:

- tenant setup steps
- app registration steps
- connector permission notes
- incident handling procedures
- recurring operational procedures

## `scripts/`

Use for:

- PowerShell helpers
- Graph query examples
- SharePoint migration helpers
- test harnesses for automation flows
